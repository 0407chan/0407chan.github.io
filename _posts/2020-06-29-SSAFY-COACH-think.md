### 코치란 무엇인가?
- 한발 앞에서 길을 안내해주는 길잡이.


### ssafy 2학기는 자기주도 학습이면 코치가 어느정도까지 도움을 주어야 할까?
- 자기주도 학습이니 혼자 하게 두어야 할까? 그럼에도 불구하고 도움을 줘야할까? 이게 코치하는 내내 가장 큰 고민이었다. 그냥 둬도 잘하는 분들이 있었고 잘하는 것처럼 보이지만 묘한 방법으로 하는 분들이 있었다.
- 물고기를 줄 것인가 물고기를 낚는방법을 가르쳐줄 것인가? 로 생각해보면 좋을 것 같다.
- 프로젝트를 하는 내내 계속 답인 물고기만 주다보면 교육생들은 물고기 받는 방식으로만 질문을 하게 될 것이다.
- 그래서 나는 물고기를 낚는 방법을 가르쳐주려고 노력을 많이 했다. 이런 경우의 경우 어떻게 검색을 하는 것이 좋은지, 먼저 검색을 해봤다면 그 검색결과와 내 검색결과가 어떻게다른지 검색 결과를 통해 확인 시켜주는 방식으로 했다.
    - 이렇게 하니 나중에는 이런이런 상황인데 검색할 키워드를 알려줄 수 있겠냐는 질문들이 들어오더라. 
- 개발자의 기본 역량이지만 간혹 지금 하는 방식으로 문제가 없이 잘 작동해서 그대로 쭉 잘못된 방식으로 하는 예시도 있었다. Git 사용방법이 그랬다.
- git flow 전략이 없이 git을 사용하고 있다던가, git 자체를 이해하지 못해서 merge할 때 어떤식으로 merge해야하는지 몰라서 강제 merge 하는 방법 등을 검색하는 모습을 봤었다. 그걸 보고 git branch 전략 강의를 만들어서 강의했다.


### 열심히 하지 않는 교육생은 어떻게 해야할까?
- 제일 어려운 문제였다. 시스템상 바꿔줄 수 없는 부분이라 참 어려웠다. 챙기자니 대충하고 외면하자니 너무 버리는것같고.. 사람 대하는 일이라 나도 상처를 받고 부담스럽고 어렵지만 그럼에도 불구하고 계속 다가가는게 필요한 것 같다.


### ssafy교육을 떠나 개발자를 위한 교육에는 어떤 것들이 들어가면 좋을까?
- ssafy교육은 아무것도 모르던 사람들이 들어와서 배우기엔 다소 빡센 부분이 있지만, 배우고 난 후 현업을 목표로 한다면 또 아쉬운점이 느껴진다. 뭔가 하는것 같지만 확실하게 손에 잡히진 않는 느낌. 


### 개발자를 위한 하농은 어떤게 있을까?
- 하농은 피아노치는 사람들이 손가락 연습을 위해 치는 연습곡 같은 개념이다. 이걸 잘한다고 모두가 피아노 고수가 되는건 아니지만, 고수중에 이걸 안한사람은 없다고 봐도 무방하다.
- 각 분야별로 이런식으로 '기본기'를 다지는 방법들이 있는데 개발자는 어떤 방식으로 이것을 훈련하는지 막연한 부분이 있었다.
- 프로젝트가 끝나서 내가 잘 하는 것 같지만 1주 2주 지나면 점점 감각이 무뎌지기 때문에 프로젝트 단위로 개발을 하는 개발자들에겐 하농이 너무 필요하다고 생각했다.
