### 함수분리

- 함수 재사용하니까 좋다
- 스크롤이 가로로 넘어가는 코드는 똥이다.

```js
array.filter(container => containersjals sadfj).every()
```

- 이런 경우

```js
array.filter(filterCheck(container)).every();

function filterCheck(container) {
  container;
  dhhdh;
}
```

- 이렇게 안쪽에 있는 함수를 외부 함수로 만들어서 호출하면 더 간편해지고 보기도 편해진다.

### if-else

```js
if (hello) {
} else {
  if (hi) {
  } else {
  }
}
```

- 이런 경우 안쪽에 있는 if else를 함수로 만들어서 넣어서 분리하는 것이 권장 된다. 하나 안에 너무 많아지면 보기가 불편하다.

### 범용 코드

- 나만의 유틸리티를 만들어가는 것을 해보자.
- 작은 함수의 경우 범용 코드로 만들 수 있는 것들은 범용으로 만들어서 유틸리티화 하자.
- 내 유틸리티가 많아지면 새로운 프로젝트 할 때 import해서 쓰면 개발속도가 매우 빨라질 수 있다.

- `util`은 정말 범용적이고, 비지니스가 아닌것 들을 이야기한다.
- 다른서비스에서도 쓰일 수 있을정도로 범용적인 것들을 util이라고 한다.

### 백엔드

- 매직넘버 넣으면 안된다.
- 변수에 수를 할당해서 그 변수를 넣어야한다.
- .emv 같은거를 많이 쓴다.
- 환경 변수를 만들어서 밖에다가 써놓고 그걸 불러서 쓰자

- hotfix가 매우 크리티컬 하다.
  - 장애가 일어나면 해야한다.
- 개발과 유지보수비율은 20:80% 이다.
- 핫픽스를 하기위한 전략을 알고 있어야한다.
- 핫픽스하려먼? 이전 버전으로 돌려야 한다.
- 태그를 달아서 이전 버전으로 가는데, 어떤 버전이 완전한지는 모른다.
  - 그렇기 때문에 기능별로 커밋을 날리는게 좋다.
  - 잘못된 커밋을 찾아내서 그 커밋에 revert를 날린다. revart는 해당 커밋에 새 커밋을 날려서 덮는거라 생각하면 된다.
- 이걸 잘하려면 커밋과 로그가 많이 많이 남아 있어야한다. 그래서 내용을 풍부하게 적는것이 매우좋다. 그러니 최대한 서버에 기록을 많이 남기자
