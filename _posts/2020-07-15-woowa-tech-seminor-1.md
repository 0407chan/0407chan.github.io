# 프론트엔드 개발자가 되기 위한 10가지 방법

발표자: 윤지수

## 1. 현장 이해하고 대응하기

- 함께 일하기
- 분석, 개발, 테스트, 배포, 피드백, 개선
- 제품과 서비스의 지속적인 운영
- 여기에서 발생하는 문제들! 이걸 어떻게 해결할까?

  - 커뮤니케이션의 어려움
  - 일의 순서 정리
  - 다양한 제약 사항
  - 품질과 일정
    - 품질을 늘리자니 일정이 쪼들리고, 일정에 맞추자니 품질이 떨어진다..
  - 형상관리
  - 테스팅의 어려움

### 이걸 연습하기위해서?

- 연습은 실전처럼!

  - 기초 프로그래밍 연습 -> 미니 프로젝트 -> 좀더 큰 프로젝트 -> 같이 할 수 있는 프로젝트
  - 프로젝트를 하면서 모르는 부분을 찾아보자
  - 인턴경험, 알바 경험 등등

- 문제 해결 능력 향상
  - 다양한 방법으로 문제를 해결하고, 회고하고,
  - 디버깅 잘 이해하기
  - 다른사람의 방법도 배우자 (페어프로그래밍 등)

## 2. 프로젝트를 통해서 JS 익히기

### 자바스크립트 공부하는법

- JavaScript 익히는 것이 중요하다
- 책과 프로젝트의 거리감 줄이기
- 프로젝트를 하면서 언어 책을 함께 공부하자
  - 이런게 prototype이구나!
  - closure 를 활용해서 이렇게 구현할 수 잇구나!
  - let, const 의 차이가 이거구나!
- 면접에서 이론을 묻는 이유는?
  - 기초지식을 충분히 알고 대응하는 개발자인지 보기 위해서!

## 3. 향상된 UX를 목표로 개발하기

- 향상돈 UX를 목표로 개발하자
- 사용자가 쓰기에 불편한이 없는 인터랙션 개발
  - 애니메이션
  - 좀더 빠르고 좀더 직관적인 UX고민, 개선
- 완벽한 코드보다, **사용성 향상작업** (성능, 매끄러움)
- CSS3, DOM, Event, Animation 제어, 비동기에 대한 이해

## 4. 함께 자라기

- 함께 공부하고
- 일감 만들어서 관리하기 issue, project, wiki 등등
- git branch 전략 세워보기
- 코드리뷰하기
- 커뮤니티 이용하기

## 5. 기초는 튼튼하게

### 기초에서 중요한 것들은?

`네트워크`

- http 개발자 도구의 네트워크 탭을 다 이해해보자.
- 웹개발은 클라이언트와 서버간의 커뮤니케이션

`백엔드 접해보기`

- node.js 중심으로 웹애플리케이션 서버와 연동해보기
- db, os, 클라우드등의 지식

`자료구조, 알고리즘`

- 시간 재기
- 쉬운 문제 많이 풀어보기
- 같은 문제 다시 풀어보기
- 다른사람 코드 보기

## 7. 양 < 질

- 로드맵 필요없다.
- 무언가를 **찐~~하게 대할 수 있는지가 더 중요하다**
- 내가 하고 있는 분야를 깊게 파고 있는가?
  - 웹사이트를 꾸준하게 만들고 있는가?
    - 좀더 사용성을 올리기 위해서 다양한 방법을 찾아보고 시도했는가?
    - 웹사이트를 꾸준이 만들고 피드백 받으며 개선했나?
    - 좀더 나은 코드품질을 위해 다양한 리팩토링을 시도하고있는가?
    - 내가얻어낸 코드를 공유하고 노하우를 공유했는가?
