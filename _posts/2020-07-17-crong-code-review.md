# 크롱 코드리뷰 및 팁

- 엔트리 포인트에서는 무조건 그 페이지의 컴포넌트를 한번에 알 수 있도록 작성하는게 좋다. 그래서 어떤 모듈을 사용하고 있는지, 그 모듈들을 어떻게 배치되어있는지 한눈에 볼 수 있도록 짜자.

- 자꾸 쓰는 함수들은 쉽게 만들어서 써보자.

```js
.lassList.remove('close")

_remove(target, className);
```

이런식으로 쉽게 볼 수 있도록 하면 좋다.

- 템플릿 티러럴로 innerHTML로 쓰는게 성능이 빠르다. 문자열로 하는게 유리하다고 한다. createElement보다 이게 좋다고 함

- 함수이름은 길어도되니 명확하게 보여주면 좋다.

- 이걸 모델에 추가하느냐? 화면에 추가하느냐? 고민해봐야한다.
  - store에 추가한건지 화면에 뿌려주려고 한건지 알 수 가 없다.
  - 이러면 뷰 렌더링 하는 곳은 모델을 지켜보면 좋다.
- 결국 vue-store 구현하면 될듯. ㅋㅋㅋㅋ 굳

- 웹팩에 개발모드랑, 프로덕션 모드가 있다. 그걸 사용해서 개발용과 배표용 따로 해보자
- 프론트엔드도 뷰와 모델을 분리하는 작업이 필요하다. 어떻게 하면 할 수 있을지 고민해보자.

## 백엔드

- 설정보다 컨벤션, 파일이름은 소문자와 `_`로 구성한다.
- url은 소문자와 `_`로 구성한다.
- 로그를 기록하자
- 디버깅을 활용하자 -> `console.log` 말고 디버깅을 활용해보자
- 모르는 미들웨어 막 쓰지 말자
- 메소드 분리, 코드 분리 잘하기
- 의미없는 util은 줄이다
- 좀더 자주 커밋, 자주 테스트, 가능하면 TDD 작성하기
- 하드코딩 및 매직넘버 사용하지 말자!
